Why There Are Fewer Fields in This Section


In this redesigned EPIC layout, the Details section has been simplified to focus only on what’s essential. Here’s why:


  • Receiving Officer is now auto-populated based on how the case was received and who made it. If EnviroLine creates the case, it’ll show as through EnviroLine. If an officer enters it, their name is recorded. Same goes for after-hours entries. There should of course be an opportunity to override this if necessary. This will now be visible in the Case Summary tab instead.
  • Source of Notification has been removed. This info should already be captured naturally in the Incident Description if it’s relevant or non-standard.
  • Legislation field has been removed — it wasn’t adding value to the case record.
  • Incident Start Date is replaced with Date Discovered. In most cases, reporters don’t know when the issue started, but they always know when they became aware of the incident.
  • Incident End Date has been removed. It’s rarely known and, if relevant, time periods should be explained in the Incident Description.
  • After Hours Incident Number will now only appear if the case came in after hours. Otherwise, it stays hidden to avoid clutter.

    • Other details, like how the case came in and background info, are now displayed on the Case Summary page instead — keeping this screen clean and focused for investigation.
